#### 3.2 Subdivisión midpoint de malla

**Función:** `_subdivideGeometry(geo)` en `avatar.js:914-974`

```
Entrada: 468 vértices, 936 triángulos (FACES canónico)
Salida:  ~1870 vértices, 3744 triángulos
```

**Algoritmo:**
1. Copiar todos los vértices existentes (posiciones + UVs)
2. Para cada arista del mesh, crear un vértice en el punto medio (posición y UV interpolados)
3. Cache de aristas con Map (`"min_max" → vertexIndex`) para evitar duplicados
4. Cada triángulo se divide en 4 sub-triángulos preservando winding order
5. Dispone la geometría original y retorna la nueva

**Decisión de diseño:** Se usa subdivisión midpoint (no Loop) porque:
- No cambia la forma de la malla (misma superficie, más triángulos)
- `computeVertexNormals()` post-subdivisión produce normals mucho más suaves
- Los vértices originales mantienen sus índices (0-467) → Gemini bumps y overlay siguen funcionando
- Loop subdivision cambiaría la forma, potencialmente distorsionando el rostro

#### 3.5 Raycasting: implementado y revertido

**Función:** `_conformToSurface(positions, N, landmarks, scaleZ)` en `avatar.js:857-903`

**Lo que hace:** Para cada uno de los 468 vértices, dispara un rayo desde (x, y, 3.0) en dirección (0,0,-1) contra las meshes del modelo GLB. Si impacta, reposiciona el vértice Z en la superficie del cráneo + offset + profundidad MediaPipe.

**Por qué se implementó:** Para que la malla facial se conformara a la curvatura real del cráneo 3D, eliminando el aspecto "plano" del posicionamiento absoluto (baseZ).

**Por qué se deshabilitó:** En la práctica, el raycasting causaba que los vértices de los bordes (frente superior, sienes, mentón) impactaran en las partes curvas del cráneo, haciendo que la malla se envolviera alrededor de la cabeza como un casco/domo transparente en lugar de una máscara facial plana.

**Estado actual:** La función existe en el código pero NO se invoca. Se reemplazó por un gradiente de profundidad radial.

#### 3.6 Gradiente de profundidad radial

**Archivo:** `avatar.js:294-301`

```javascript
const ndx = (lm.x - faceCxN) / (faceWN * 0.5);
const ndy = (lm.y - faceCyN) / (faceHN * 0.5);
const dw  = Math.max(0, 1.0 - Math.sqrt(ndx * ndx + ndy * ndy) * 0.6);

positions[i * 3 + 2] = fz * scaleZ * dw + baseZ;
```

**Qué hace:** Calcula la distancia normalizada de cada vértice al centro del rostro. Vértices centrales (nariz, ojos) reciben profundidad 3D completa (`dw ≈ 1.0`). Vértices periféricos (sienes, frente alta, mentón lateral) reciben profundidad atenuada (`dw → 0`), acercándose a `baseZ` (plano).

**Valores del gradiente:**
| Posición | edgeDist | dw (peso) | Efecto |
|---|---|---|---|
| Nariz (centro) | 0.0 | 1.0 | Profundidad 3D completa |
| Ojos | ~0.4 | 0.76 | 76% de profundidad |
| Mejillas | ~0.6 | 0.64 | 64% de profundidad |
| Borde oval | ~1.0 | 0.40 | 40% de profundidad |
| Esquinas extremas | ~1.4 | 0.16 | Casi plano |

**Resultado:** La malla tiene volumen 3D pronunciado en el centro (nariz sobresale, cuencas de ojos se hunden) pero se aplana gradualmente hacia los bordes, donde el alpha map la funde con el maniquí.

## Decisiones técnicas importantes

### Por qué NO usar raycasting para posicionamiento Z

El raycasting contra la superficie del cráneo es conceptualmente correcto para conformar la malla. En la práctica falla porque:

1. La malla de MediaPipe cubre más allá del rostro visible (frente alta, sienes)
2. Los rayos en esas zonas impactan la curvatura del cráneo (que va hacia atrás)
3. El resultado es una malla que envuelve la cabeza como un casco
4. El alpha map no puede ocultar este efecto porque la geometría ya está deformada en 3D

El posicionamiento absoluto (baseZ + gradiente radial de MediaPipe Z) funciona mejor porque:
- Produce una máscara facial "pseudo-plana" que se integra con el maniquí
- El alpha map difumina los bordes limpiamente
- La profundidad de MediaPipe da suficiente volumen 3D (nariz, ojos, mejillas)
- No hay riesgo de envolver la cabeza

### Por qué subdivisión midpoint y no Loop

Loop subdivision suaviza la geometría desplazando vértices, lo que cambiaría la forma del rostro (landmarks ya no estarían en posiciones correctas). Midpoint subdivision solo agrega vértices sin mover los existentes. El suavizado visual viene de `computeVertexNormals()` con más triángulos → normals más interpoladas.

### Por qué RoomEnvironment y no un HDRI externo

RoomEnvironment es un addon incluido en Three.js que genera un estudio de iluminación procedural. Ventajas:
- Sin archivo externo (no hay `.hdr` que cargar)
- Se genera en GPU instantáneamente
- Produce reflejos suaves y neutros (ideal para piel)
- El segundo parámetro (0.04) controla la intensidad del blur

## Convenciones críticas a recordar

1. **Todas las texturas usan `flipY = false`** (photo, alpha, normal). Si alguna usa `true`, el UV mapping se rompe.

2. **Normal map siempre `LinearSRGBColorSpace`**. Si cambia a `SRGBColorSpace`, el relieve se deforma.

3. **Z de MediaPipe es opuesto:** `lm.z < 0` = más cerca de cámara = `+Z` en Three.js → negarlo siempre.

4. **`transparent: true` es obligatorio** para que `alphaMap` funcione.

5. **Config.js está en `.gitignore`** — nunca commitear.

6. **Los 468 vértices originales mantienen índices 0-467 tras subdivisión**. Gemini bumps y overlay dependen de esto.

7. **`scene.environment`** provee el env map a TODOS los materiales PBR automáticamente. No necesita `material.envMap`.

8. **`_conformToSurface` existe pero NO se invoca** — causa efecto domo. Usar gradiente radial en su lugar.

## Arquitectura de la malla facial v7.0

```
468 landmarks (lm.x, lm.y, lm.z)
         │
         ▼
    XY → escala absoluta (TARGET_FACE_W=1.75, TARGET_FACE_H=2.25)
    Z  → (-lm.z) * scaleZ * depthWeight + baseZ (0.28)
         │  depthWeight: radial 1.0 (centro) → 0 (bordes)
         │
         ▼
  BufferGeometry + FACES[2808] (triangulación canónica)
         │
         ▼
  _subdivideGeometry() → 3744 triángulos (~1870 vértices)
         │
         ▼
  computeVertexNormals() → normals suaves
         │
    photoTex ─── map           (SRGB, flipY=false)
    alphaTex ─── alphaMap      (oval encogido 12%, blur r=12, 4 pasadas)
    normalTex ── normalMap     (Sobel STR=3.0, LinearSRGB, flipY=false)
         │
    MeshPhysicalMaterial (transparent, clearcoat=0.12, sheen=0.5, envMapIntensity=0.8)
         │
    THREE.Mesh → head.add(mesh) → renderOrder=1
         │
    [Gemini async] → bumps Z (solo índices 0-467) + roughness + normalScale + escala
         │
    scene.environment ← PMREMGenerator(RoomEnvironment) → reflejos realistas
```
